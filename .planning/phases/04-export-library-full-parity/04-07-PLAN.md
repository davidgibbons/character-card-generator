  </what-built>

  <how-to-verify>
Start the dev server: `cd /Users/dgibbons/git/character-card-generator && npm run dev`

Then verify each item:

**1. CharacterEditor RPG layout**
- Generate a character (any concept)
- Verify: portrait slot appears top-right (square, dashed border placeholder)
- Verify: Name, Tags, Creator Notes appear top-left
- Verify: Character/Lorebook subtab bar appears below the header
- Verify: Description, Personality, Scenario, First Message, Message Example in body
- Verify: "Advanced Settings" collapsible at bottom — expand it — System Prompt is inside

**2. Image generation + upload (PARITY-03)**
- With a character generated: click "Generate Image"
- Verify: portrait slot shows pulsing animation during generation
- Verify: after generation, portrait fills with the generated image
- Verify: button label switches to "Regenerate Image"
- Also test: click "Upload Image", select a local image file
- Verify: uploaded image fills the portrait slot

**3. Export buttons (PARITY-04, REACT-11)**
- Click "Save Card" — verify success: button shows "Saved" for ~1.5s, then reverts
- Click "Download JSON" — verify: browser downloads a .json file with character data
- Click "Download PNG" — verify: browser downloads a .png file (open it — should be a valid PNG)
- Verify: all three buttons are disabled when no character exists

**4. Library drawer (REACT-07, PARITY-05, STATE-04)**
- Click the 📖 book icon in the header to open library
- Verify: drawer slides in from right
- Verify: your saved card appears in the list with name, date, and quality badge (if any)
- Verify: search input filters the list by name
- Click the card row — verify: "Discard Changes?" dialog appears if character has unsaved changes
- Verify: clicking a card loads it into CharacterEditor

**5. Card history + diff (REACT-12)**
- Save a card, edit a field, save again (to create 2 commits)
- Open library, click 📋 History on the card
- Verify: history modal shows commit list
- Click two commits, click "Show Diff"
- Verify: DiffView shows before/after for changed fields

**6. Lorebook tab (REACT-08, PARITY-09)**
- Click "Lorebook" subtab in CharacterEditor
- Click "Generate Lorebook" — verify: entries appear after generation
- Click an entry row to expand — verify: Keys, Content, Comment, Priority fields visible
- Lock an entry (click 🔒) — lock button gets yellow border
- Click "Regenerate Lorebook" — verify: confirmation dialog appears
- Confirm replacement — verify: locked entry is preserved, others replaced

**7. SillyTavern sync (REACT-09, PARITY-06)**
- Open Settings → scroll to SillyTavern section
- Enter your ST URL and password (or leave blank to verify UI structure)
- Verify: "List Characters", "Push Character" buttons are present
- If ST is available: test push and pull

**8. @mention autocomplete (D-19)**
- Open Settings → close
- Click into the concept textarea in CreatePanel
- Type `@` — verify: autocomplete dropdown suggests saved card names
  </how-to-verify>
