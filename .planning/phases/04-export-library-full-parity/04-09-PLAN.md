**Gap source:** 04-VERIFICATION.md GAP-02

**Two bugs in one field:**
- (a) Display bug: `draft.api.text.timeout` stores milliseconds (e.g. 30000). Input `value={draft.api.text.timeout}` shows 30000. Should show 30.
- (b) Edit bug: `onChange` calls `updateDraft('api.text.timeout', parseInt(e.target.value) || 180000)`. When user clears the field (`e.target.value = ""`), `parseInt("")` is `NaN`, so `NaN || 180000 = 180000` — field immediately resets to 180000. User cannot type a new value.

**Fix pattern:** Local edit state for each timeout field that stores the raw string the user is typing. On blur (or when value is a valid integer), commit `value * 1000` to draft.

**Draft pattern** (from Phase 2): Settings modal opens with a deep-copy of configStore as `draft`. User edits draft locally. Save button calls `configStore.set()` for each draft path. `updateDraft(path, value)` prop writes to draft. This pattern is already working for all other fields.

**Two timeout fields in the component:**
1. Text API: `draft.api.text.timeout`
2. Image API: `draft.api.image.timeout`

Both need the same fix.

    value={textTimeoutEdit}

    onChange={(e) => {

      if (!isNaN(secs) && secs > 0) {
