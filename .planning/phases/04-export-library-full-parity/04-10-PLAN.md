**Gap source:** 04-VERIFICATION.md GAP-03

**Root cause:** `react-mentions` `MentionsInput` registers keydown listeners on the underlying textarea to handle suggestion list navigation (ArrowUp/ArrowDown to move between suggestions, Enter/Tab to select). These listeners fire even when the dropdown is closed, consuming the events before they reach the browser's default textarea cursor-movement behavior.

**react-mentions API:** `MentionsInput` does not expose a prop to disable key capture when closed. The workaround is to add an `onKeyDown` handler that intercepts ArrowUp/ArrowDown and calls `e.stopPropagation()` when the suggestion list is NOT open, allowing the browser's native behavior. However, `MentionsInput` does not expose an "is open" state prop directly.

**Alternative approach (simpler):** Check if there is currently a query being typed after `@` by tracking suggestion state. But react-mentions does not expose a simple "dropdown open" boolean to the parent.

**Pragmatic fix:** Use a `ref` on the underlying input to track whether suggestions are showing. react-mentions passes `onSuggestionsPopupId` via aria attributes — when the popup is open, `aria-expanded="true"` is set on the input. Read `inputRef.current.getAttribute('aria-expanded') === 'true'` in the onKeyDown handler.

**Implementation:** Pass a custom `onKeyDown` to `MentionsInput`. In the handler, if the key is `ArrowUp` or `ArrowDown` AND `aria-expanded` is NOT `"true"`, call `e.stopPropagation()` to prevent react-mentions from consuming the event. Do NOT call `e.preventDefault()` — only `stopPropagation()`, so the browser still processes the key for cursor movement.

**Note on react-mentions internals:** The library adds its own keydown handler via `addEventListener` on the underlying textarea (not via React's synthetic events). `stopPropagation()` on the synthetic React event will not stop the native listener. Instead, use a native `addEventListener` approach or — simpler — use the `inputRef` to add a capture-phase listener.

**Correct approach using capture phase:**
```js
useEffect(() => {
  const textarea = inputRef.current?.querySelector('textarea');
  if (!textarea) return;
  function handleKeyDown(e) {
    if ((e.key === 'ArrowUp' || e.key === 'ArrowDown') &&
        textarea.getAttribute('aria-expanded') !== 'true') {
      e.stopPropagation();
      // Do NOT preventDefault — let browser move cursor
    }
  }
  textarea.addEventListener('keydown', handleKeyDown, true); // capture phase
  return () => textarea.removeEventListener('keydown', handleKeyDown, true);
}, []);
```

This uses capture phase (`true` as third arg) so our handler runs before react-mentions' bubble-phase handler, and `stopPropagation()` prevents the library from seeing the event when no dropdown is open.
