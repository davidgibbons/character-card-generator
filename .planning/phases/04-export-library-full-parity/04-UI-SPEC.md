## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, lock button padding (FieldRow: `padding: 4px 8px`) |
| sm | 8px | Compact inline gaps, tag gaps in card list rows |
| md | 16px | Default form group spacing, drawer body padding unit |
| lg | 24px | Section padding in panels, drawer/modal body padding |
| xl | 32px | Major layout gaps between header/body sections |
| 2xl | 48px | Large section breaks (e.g. empty state vertical centering) |
| 3xl | 64px | Page-level spacing (not typically used in drawer/modal surfaces) |

Exceptions:
- Library drawer width: `min(400px, 90vw)` — fixed from globals.css, not overridden
- Portrait image slot: `~25-30%` of header section width (RPG sheet layout, D-01)
- Modal width: `min(760px, 100%)` — existing Modal.jsx constraint, reused for history/diff
- Drawer slide transition: `transform 250ms ease` — matches existing `.library-drawer` in globals.css
- Touch target minimum: 44px height on all action buttons (per ActionBar stopBtn precedent)

## Typography

Source: globals.css `:root`, FieldRow.module.css, CharacterEditor.module.css, ActionBar.module.css

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14.7px (0.92rem) | 400 (regular) | 1.5 |
| Label | 15.4px (0.96rem) | 600 (semibold) | 1.2 |
| Heading | 16.8px (1.05rem) | 600 (semibold) | 1.2 |
| Display | clamp(24.8px, 2.3vw, 40.8px) | 700 (bold) | 1.0 |

Notes:
- 0.92rem body = ~14.7px at 16px root; matches FieldRow textarea, CharacterEditor emptyBody, ActionBar errorMsg, globals .label
- 0.96rem label = ~15.4px at 16px root; matches FieldRow .label
- 1.05rem heading = ~16.8px at 16px root; matches library-drawer-header h2, Modal .title
- Display is the app title (clamp in globals.css .title) — not used in Phase 4 new components
- All new components in Phase 4 use the same 0.92rem/0.96rem/1.05rem scale — no new sizes introduced
- Subtab labels (e.g. Character / Lorebook tab): 0.85rem at weight 600 (matches globals.css .subtab-btn)

## Color

Source: globals.css `:root` CSS custom properties

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--bg-page)` = `#f3f7fb` | Page background, ActionBar sticky background |
| Secondary (30%) | `var(--surface)` = `rgba(255,255,255,0.8)` / `var(--surface-strong)` = `#ffffff` | Panels, drawer body, card rows, modal body |
| Accent (10%) | `var(--accent)` = `#0e7a8a` | Active subtab underline, active badge, focus rings, quality score badge |
| Destructive | `var(--error)` = `#c93d4a` | Delete card button, error state copy, destructive confirmation buttons only |

Accent reserved for:
- Active Lorebook subtab underline (matches existing `.subtab-btn.active` pattern)
- Quality score badge text/border on card list rows
- Focused input outline (matches existing globals.css input focus-visible pattern)
- Tab badge pulse indicator (`.tab-badge` in globals.css)

Secondary semantic colors (not accent, not destructive):
- `var(--warning)` = `#cf8a00` — locked field border and locked lorebook entry border (reuses FieldRow locked pattern)
- `var(--success)` = `#1f9d66` — "Saved" confirmation feedback after Save card action
- `var(--text-secondary)` = `#51667c` — Card list metadata (date, tags), empty state headings
- `var(--text-soft)` = `#7d8ea1` — Empty state body copy, placeholder text

Dark theme: all color tokens have dark overrides already defined in globals.css and component CSS Modules. Phase 4 components follow the same `:global([data-theme="dark"]) .selector` override pattern.

## Layout Specifications

### CharacterEditor RPG Sheet (D-01, D-02)

```
┌─────────────────────────────────────────────────────┐
│  HEADER SECTION (two-column)                         │
│  ┌──────────────────────────┐  ┌──────────────────┐  │
│  │ Left ~70-75%             │  │ Right ~25-30%    │  │
│  │ Name field               │  │ Portrait image   │  │
│  │ Tags field               │  │ slot / avatar    │  │
│  │ Creator Notes field      │  │                  │  │
│  │ [Upload] [Generate img]  │  │                  │  │
│  └──────────────────────────┘  └──────────────────┘  │
├─────────────────────────────────────────────────────┤
│  CHARACTER / LOREBOOK subtab bar                     │
├─────────────────────────────────────────────────────┤
│  BODY: full-width large fields OR lorebook entries   │
│  Description, Personality, Scenario, First Message  │
│  Message Example, System Prompt                     │
├─────────────────────────────────────────────────────┤
│  ▶ Advanced Settings (collapsible)                  │
│    System Prompt (if moved here), Creator Notes alt │
└─────────────────────────────────────────────────────┘
```

Advanced Settings collapsible (D-02): System Prompt field lives here by default. Creator Notes stays in the compact header if it fits; moves to Advanced Settings if header becomes crowded. Claude's discretion on final field placement.

### Portrait Image Slot (D-03, D-04, D-05)

- Dimensions: fills right column of header section; aspect ratio `1/1` (square avatar)
- Before generation: muted placeholder with dashed border (`var(--border-strong)`), centered icon
- After generation: image fills slot with `object-fit: cover`, `border-radius: var(--radius-md)`
- Upload button label: "Upload Image" (secondary style button)
- Generate button label: "Generate Image" (idle) / "Regenerate Image" (after image exists)
- Both buttons in left column, below compact fields

### Library Drawer

- Position: `fixed`, right edge, full viewport height (reuses globals.css `.library-drawer` shell)
- Width: `min(400px, 90vw)`
- Open animation: `transform translateX(100%) → translateX(0)`, `250ms ease` (matches existing)
- Backdrop: `rgba(14,23,37,0.35)`, `opacity 0 → 1`, `250ms ease` (matches existing)
- Search input: full-width at top of drawer body
- Card row height: auto (content-driven); minimum touch target 44px
- Tags in card row: flex-wrap, up to 3 tags then truncate with count badge ("…+N")
- Quality score badge: accent-colored pill (if present from API response)

### ActionBar Export Buttons (D-06, D-07, D-08)

Button group added after existing Generate/Evaluate/Revise buttons:
- "Save" — primary-style, only active when `character != null && !isStreaming`
- "JSON" — secondary-style, same active condition
- "PNG" — secondary-style, same active condition

## Interaction States

### ActionBar Buttons — State Machine

| Button | Idle | Streaming | No Character |
|--------|------|-----------|--------------|
| Generate | enabled | disabled | enabled |
| Stop | hidden | enabled | hidden |
| Evaluate | enabled | disabled | disabled |
| Revise | enabled | disabled | disabled |
| Save | enabled | disabled | disabled |
| Download JSON | enabled | disabled | disabled |
| Download PNG | enabled | disabled | disabled |

### Library Card Row States

| State | Visual |
|-------|--------|
| Default | white/surface background, subtle border |
| Hover | `var(--accent-soft)` background tint |
| Active/Loading | skeleton shimmer (same `var(--border)` pattern) |

### Lorebook Entry Row States

| State | Visual |
|-------|--------|
| Collapsed | single-line summary, expand chevron right |
| Expanded | full inline form, collapse chevron down |
| Locked | `var(--warning)` border tint (matches FieldRow.locked) |
| Enabled toggle off | disabled visual, reduced opacity on content |

### Loading States

- Image generation in progress: portrait slot shows pulsing placeholder (CSS animation, opacity 0.4→0.8 loop, 1.2s)
- Lorebook generation in progress: "Generating…" button label, spinner icon, entries list shows skeleton rows
- ST List Characters: button shows "Loading…", list area shows skeleton rows
- Card list loading: skeleton rows in drawer (3 placeholder rows)
