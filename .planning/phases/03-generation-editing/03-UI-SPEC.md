## Spacing Scale

All values follow the existing rem-to-pixel mapping at 16px root (1rem = 16px).
Declared values align with 4-point multiples:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (0.25rem) | Icon-to-label gaps, lock button internal padding |
| sm | 8px (0.5rem) | Compact row gaps, inline button spacing |
| md | 16px (1rem) | Default form-group margin, section padding |
| lg | 24px (1.5rem) | Panel section vertical padding |
| xl | 32px (2rem) | Inter-panel layout gaps |
| 2xl | 48px (3rem) | Major section breaks (unused in this phase) |
| 3xl | 64px (4rem) | Page-level spacing (unused in this phase) |

Exceptions:
- Stop button (`#stop-btn`): minimum touch target 44px height — source: globals.css line 381
- Field lock button padding: 4px 8px (0.25rem 0.5rem) — compact toggle, not a primary tap target
- ActionBar gap: 8px (0.5rem) between button group and ProgressBar — preserves compact ActionBar

Source: globals.css spacing audit.

## Typography

| Role | Size | Weight | Line Height | Font |
|------|------|--------|-------------|------|
| Body / form input | 14.7px (0.92rem) | 400 regular | 1.5 | Sora / IBM Plex Sans |
| Label (field name, section title) | 15.4px (0.96rem) | 600 semibold | 1.2 | Sora / IBM Plex Sans |

| Small / metadata (lock btn, hint text) | 11.5px (0.72rem) | 400 regular | 1.3 | Sora / IBM Plex Sans |

Rules:
- Exactly 2 font weights in use: 400 (regular) and 600 (semibold). Weight 700 is reserved for the app title (Header) — not used in Phase 3 components.
- Stream output MUST use the monospace stack. No exceptions.
- Label size (0.96rem / 600) applies to: field names in FieldRow, section heading "Evaluation Feedback", revision instruction label.
- Body size (0.92rem / 400) applies to: all textarea content, concept input, name input, POV selector.

Source: globals.css `.label` (line 293), `.stream-box` (line 484), `.character-field h3` (line 519), `.btn-field-lock` (line 1547).

## Color

All colors come from the CSS custom property system. Components reference tokens, never raw hex values.

| Role | Light value | Dark value | Token |
|------|-------------|------------|-------|
| Dominant (60%) — page background | #f3f7fb | #111827 | `--bg-page` |
| Secondary (30%) — panels, cards, inputs | rgba(255,255,255,0.8) / #ffffff | rgba(22,32,48,0.92) / #1a2637 | `--surface` / `--surface-strong` |
| Accent (10%) | #0e7a8a | #1aaabf | `--accent` |
| Destructive | #c93d4a | #e05060 | `--error` |
| Warning / locked state | #cf8a00 | #e0a020 | `--warning` |

Accent (`--accent`) is reserved for:
1. Generate button background (`.btn-primary` gradient uses `--accent` / `--accent-strong`)
2. Active tab indicator (TabBar — existing)
3. Link/hover state on outline buttons (border + text color on hover)
4. ProgressBar fill
5. Focus ring on textarea/input elements

Destructive (`--error`) is reserved for:
1. Stop button background (`.btn-stop`)
2. Error state text and border highlight

Warning (`--warning`) is reserved for:
1. Field lock button border when locked (`btn-field-lock.locked`)
2. Locked textarea border tint (`rgba(207,138,0,0.4)`)
3. Locked field background tint (`color-mix(in srgb, --warning 5%, --surface-muted 95%)`)

Surface muted (`--surface-muted`) is the default background for:
- Eval feedback block (read-only, clearly not editable)
- Locked field textareas

Source: globals.css `:root` block, `.btn-field-lock` (line 1560), `.character-field.field-locked` (line 1565).

## Interaction States

### Right panel switching

The right panel renders exactly one of two views — never both simultaneously:

| Condition | Panel renders |
|-----------|---------------|
| `character === null` AND `streamText === ''` | Empty editor state (centered message) |
| `isGenerating === true` | StreamView (streaming text, auto-scroll) |
| `isGenerating === false` AND `streamText !== ''` AND `character === null` | StreamView (parse failed — shows raw text with error note) |
| `character !== null` | CharacterEditor (field list) |

Transition: no animation — instant swap. The ProgressBar in ActionBar provides the in-progress signal.

### Field lock toggle

- Click lock button: toggles locked state in `useGenerationStore.lockedFields` Set
- Locked: textarea border becomes warning-tinted, background becomes muted-warning tint, lock icon full opacity, 1px `--warning` border on button
- Unlocked: textarea returns to normal, lock button opacity 0.4
- Locked fields cannot be edited — textarea is `disabled`

### Streaming auto-scroll

StreamView scrolls to bottom on every chunk. If the user manually scrolls up (scrollTop < scrollHeight - clientHeight - 50px threshold), auto-scroll pauses. It resumes when the user scrolls back to bottom or when a new generation starts.

### react-mentions dropdown trigger

- Triggered by typing `@` anywhere in the concept textarea
- Shows suggestion list inline below cursor position (react-mentions default positioning)
- Suggestion items: card name (0.88rem, `--text-primary`) — full library population deferred to Phase 4 (Phase 3: skeleton with empty list or static placeholder cards)
- Selecting a suggestion inserts `@CardName` pill
